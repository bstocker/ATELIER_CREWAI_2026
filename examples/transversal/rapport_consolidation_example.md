## 1. SYNTHÈSE EXÉCUTIVE

### Diagnostic consolidé

L'analyse comparative des trois modules métier du SIH (Admission, Pharmacie, Bloc Opératoire) révèle un **désalignement systémique critique** affectant la chaîne de valeur complète du patient hospitalier : de son admission jusqu'à son facturation et suivi post-intervention.

**Niveau global de désalignement : CRITIQUE**

Le SIH fonctionne selon un **système dual irréductible** : d'un côté, des processus formels prescrivant exécution séquentielle, validations explicites et traçabilité centralisée ; de l'autre, une réalité opérationnelle reposant sur des contournements systémiques (papier, appels téléphoniques, cahiers locaux, messagerie informelle, doubles saisies) pour absorber les tensions entre charge variable, ressources limitées et urgences intercurrentes.

Ce désalignement n'est **pas un problème de compétence utilisateur**. C'est un problème de **gouvernance système** : absence d'arbitrage formel sur les tensions (urgent vs. administratif, robustesse vs. flux), absence de responsable transverse garant des flux bout-en-bout, absence de paramétrage SIH supportant les cas réels, culture d'acceptation de l'imparfait normalisée.

## 3. DÉSALIGNEMENTS RÉCURRENTS CONSOLIDÉS

### 3.1 Désalignement #1 : Double saisie systématique (CRITIQUE)

**Manifestation consolidée** :
- **Admission** : Formulaire accueil papier → ressaisie SIH retardée (15-30 min/patient, 400-800 h/an)
- **Pharmacie** : Prescription papier scannée + demande SIH parallèle sans réconciliation (alerte "Attention double saisie")
- **Bloc** : Évaluation infirmier papier → ressaisie SIH ; sortie bloc confirmée + service saisit retour = deux événements divergents

**Cause racine convergeante** :
- **Technique** : SIH absence terminal accueil, trop lent contexte urgence, pas de saisie mobile rapide, interfaces lourdes
- **Organisationnelle** : Manque de postes SIH, interruptibilité élevée, charge volontaire de saisie n'est pas absorbable
- **Humaine** : Équipes adaptent en saisissant papier d'abord (buffer sécurité) puis ressaisissant ultérieurement

**Impact quantifié** :
- Charge supplémentaire : 30-60 min/patient supplémentaire
- Erreurs transcription : +200% (OCR imparfait, ressaisie manuelle)
- Taux rejet facturation : +5-10pp (données divergentes)

**Risques** :
- Sécurité patient : Erreurs identité (doublons IPP, divergences papier/SIH)
- Réglementaire : Traçabilité fragmentée, dossiers non-auditables
- Operationnel : Surcharge administrative 220-430 k€/an

### 3.2 Désalignement #2 : Validations et traçabilité tardives (CRITIQUE)

**Manifestation consolidée** :
- **Admission** : Contrôles F20/F21 réalisés juste avant facturation (j+5-6) ; anomalies découvertes trop tard corriger
- **Pharmacie** : Réception rapide entrée de stock (j+0) + saisie détaillée reportée (j+3-7) ; écart stock théorique/réel persistant
- **Bloc** : Actes validés fin journée ou jours après intervention (j+1-3) ; compte rendu opératoire rédigé lendemain ; validation sans inspection approfondie

**Cause racine convergeante** :
- **Fonctionnelle** : Processus conçu pour "validation finale avant facturation" au lieu de "validations progressives continues"
- **Organisationnelle** : Responsables validation (médecin, pharmacien) surchargés ; pas d'alerte proactive ; pas de SLA maxi
- **Technique** : Pas de checkpoints intermédiaires SIH ; pas d'alertes automatis ; traçabilité décalée

**Impact quantifié** :
- Délai détection anomalies : 5-6 jours vs. 1-2 jours possibles
- Corrections urgentes concentrées fin journée : charge accrue, erreurs augmentées
- Facturation retardée : 5-15 jours vs. 1-3 jours cible

**Risques** :
- Sécurité patient : Validation sans inspection approfondie, complication documentée tardivement, divergence prescription/exécution
- Réglementaire : Horodatages décalés, signatures non contemporaines, dossier incomplet audit
- Opérationnel : Clôture dossier retardée, facturation impossible jusqu'à validation, retard recette 3-7 jours

## 6. DÉPENDANCES INTER-MODULES CRITIQUES

### Chaîne critique admission → imputation → validation → facturation

```
FLUX PRINCIPAL (Patient admission → soins → facturation) :

├─ ADMISSION F11 (créer admission incomplète possible)
│   ↓
├─ PROBLÈME : Champs obligatoires vides → imputation impossible
│   ↓
├─ ADMISSION F16 (complétude) → DÉPENDANCE : qui trigger ? SLA ?
│   ├─ Si F16 non-activée : arrière-office improvise régularisations informelles (non-tracées)
│   └─ Si F16 activée : complétude j+1 ok ; imputation peut procéder
│       ↓
├─ PHARMACIE F17/F18 (imputation) → DÉPENDANCE : quand déclencher ?
│   ├─ Si admission incomplète : imputation partielle/impossible
│   ├─ Si pharmacie dispense avant imputation : décalage j+3-7 (imputation rétroactive)
│   └─ Si admission complète + imputation immédiate : j+0 ok
│       ↓
├─ ADMISSION F20/F21 (contrôles) → DÉPENDANCE : timing ?
│   ├─ Si contrôles juste avant facturation (j+5-6) : anomalies découvertes tard
│   ├─ Si contrôles j+0 : détection précoce, correction rapide
│   └─ Si actes non-validés : contrôles incomplets
│       ↓
├─ BLOC (validation actes) → DÉPENDANCE : quand ?
│   ├─ Si validation fin journée/j+1 : compte rendu tard, actes validés sans inspection
│   └─ Si validation j+0 soir : facturation j+1 possible
│       ↓
├─ ADMISSION F22 (facturation) → DÉPENDANCE : données complètes ?
│   ├─ Si données manquantes : facturation bloquée
│   ├─ Si données complètes : facturation j+1-3 ok
│   └─ Si actes non-validés : facturation impossible
│       ↓
└─ RECOUVREMENT (trésorerie)
    └─ Si facturation j+5-15 : retard recouvrement = impact trésorerie
```

## 7. ANALYSE CAPACITANT

### 7.1 Transparence interne (capacité utilisateurs comprendre localement tâches, règles, écrans)

**État actuel : DÉGRADÉ (niveau 2/5)**

**Admission** :
- Accueil comprend : "créer admission vite, quelqu'un arrangera après"
- Arrière-office comprend : "corriger admissions incomplètes ad hoc"
- Aucun ne comprend le flow bout-en-bout ou impact son action sur aval
- Règles implicites nombreuses (urgence = complétude partielle OK ? Quand ?)

**Pharmacie** :
- Pharmacien comprend validation = sa responsabilité
- Préparateur comprend réception = sa responsabilité
- Aucun n'a BPPH articles mémorisés ni critères validation formels en tête
- Stupéfiants supposés "gérés" (registre papier) mais processus SIH? Implicite

**Bloc** :
- Chirurgien comprend "valider ses actes"
- IBODE comprend "documenter per-op"
- Check-list HAS ? Règles implicites équipe ; variations par jour
- Personne ne sait check-list exige documentation écrite HAS

**Recommandations** :
1. Documenter Golden Process formel (BPMN, RACI, règles explicites) → transparence interne améliorée
2. Former équipes 2-3 jours chacune : processus + rôles + obligations réglementaires
3. Afficher procédures en salle (pharmacie, bloc, accueil)

**Impact : Transparence interne peut passer 2→3.5/5 avec documentation + formation.**

---

### 7.2 Transparence globale (capacité acteurs comprendre flux, interdépendances, impact leurs actions autres services)

**État actuel : TRÈS DÉGRADÉ (niveau 1/5)**

**Admission** :
- Accueil ne sait pas admission incomplète cause 30 min arrière-office/patient
- Arrière-office ne sait pas ses corrections tardives bloquent imputation pharmacie
- Aucun ne comprend cascade facturation retardée = impact trésorerie

**Pharmacie** :
- Pharmacie dispense, ignore impact timing imputation sur facturation (j+3-7 vs. j+0)
- Services cliniques commandent, ignorent impact commande orale sur traçabilité

**Bloc** :
- Bloc valide tard, ignore impact sur facturation retardée (j+5-15)
- Planification orale, ignore appels répétés cause stress patient + inefficience

**Cause** : Pas de tableau de bord transverse ; pas de communication feedback ; chaque service mesure sa métrique (accueil : temps, arrière-office : précision) sans visibilité impact chaîne.

**Recommandations** :
1. Créer tableau de bord transverse visible (% admissions complétude j+0/j+1, délai imputation, délai facturation, taux rejet)
2. Réunions de coordination hebdomadaires (10 min) : alertes SLA, feedback impact
3. Inclure "coût inaction" dans communication (ex : "non complétude admission coûte 50 k€/an arrière-office")

**Impact : Transparence globale peut passer 1→2.5/5 avec tableau de bord + communication.**

---

### 7.3 Flexibilité (capacité SIH et organisation absorber contraintes terrain, permettre ajustements maîtrisés)

**État actuel : TRÈS FAIBLE (niveau 1.5/5)**

**SIH** :
- Admission F11 crée sans validation → contournement non-encadré
- Pharmacie pas module stupéfiants → improvisation papier
- Bloc pas mode check-list simplifié urgence → contournement orale
- Pas mode dégradé formel (papier SIH tombe en panne) → adaptation locale anarchique

**Organisation** :
- Urgence vs. administratif tension non-arbitrée → chacun décide localement
- Absence ressources complétude (arrière-office absorbe ad hoc) → pas scalabilité
- Pas protocole exception identitovigilance (absence pièce) → contournement évalué par infirmier chaque fois

**Cause** : SIH paramétré pour cas normal (complétude immédiate, validation écrite, synchronisation). Réalité ≠ cas normal (urgence, panne, ressources). Pas de "variant" process formalisé.

**Recommandations** :
1. Paramétrager "variantes formelles" SIH : urgence (admission minimale + complétude j+1), panne (papier structuré + resaisie j+0/j+1)
2. Créer "protocole urgence" formel : quand peut-on dévier (quels conditions?) ; qui autorise?
3. Affecter ressources dédiées (FTE complétude, FTE gestion mode dégradé) pour absorption contraintes

**Impact : Flexibilité peut passer 1.5→3/5 avec variantes SIH + protocoles urgence.**

---

### 7.4 Réparation (capacité corriger rapidement erreurs, dysfonctionnements, blocages)

**État actuel : FAIBLE (niveau 2/5)**

**Admission** :
- Erreur identité découverte tard (j+5, trop tard corriger pré-facturation) → dossier diverge papier/SIH
- Admission incomplète bloque imputation → arrière-office improvise correction sans traçabilité
- Facturation retardée découvert juste avant deadline → correction urgente chaos

**Pharmacie** :
- Stock écart découvert inventaire seulement (j+365 après) → impossible retracer cause
- Dispensation non-imputée découverte pré-facturation → correction urgente, risque omission
- DMI sans traçabilité découvert incident patient → impossible signaler fournisseur

**Bloc** :
- Acte non validé découvert pré-facturation → urgent validation sans inspection approfondie
- Check-list incomplète découvert rétrospectivement (complication) → responsabilité floue
- Planning glissant découvert constant → appels répétés, pas correction systématique

**Cause** : Problèmes détectés par incident (post facto), pas prévention (proactive). Absence checkpoints intermédiaires. Responsables flous = escalade difficile.

**Recommandations** :
1. Implémenter alertes SIH proactives (j+0/j+1 problèmes détectés pas j+5 après incident)
2. Créer responsable unique par flux (Manager Circuit) → escalade immédiate, correction rapide
3. Mode triage incident (niveau 1 = bloque patient ? → urgent ; niveau 2 = administrative ? → standard)

**Impact : Réparation peut passer 2→3.5/5 avec alertes + responsable garant + escalade formelle.**

---

### 7.5 Dialogue capacitant (capacité métiers/DSI/management/référents SIH expliciter écarts, arbitrer collectivement, co-construire)

**État actuel : FRAGMENTÉ (niveau 2/5)**

**Admission** :
- Réunion "admission" absente historiquement
- Pas espace formalisé explicitant tensions accueil ↔ arrière-office
- Arbitrage urgent/administratif laissé à chacun

**Pharmacie** :
- Réunion "pharmacie + services cliniques" inexistante
- Aucun espace explicitant tension urgence ↔ validation BPPH
- Commandes informelles = adaptation locale sans escalade

**Bloc** :
- Réunion "bloc + unité" inexistante
- Planning ignoré = personne ne débat "peut-on honorer planning réaliste ?"
- Appels répétés = symptôme adaptation locale à problème non-discuté

**Cause** : Pas de gouvernance formelle multi-disciplinaire. Chaque service "fait son job", pas de espace collectif débattre tensions, co-construire solutions.

**Recommandations** :
1. Créer Commission urgence/administratif (mensuelle CODIR) : arbitrer tensions formellement
2. Réunion coordination hebdomadaire inter-services (10 min) : alertes, feedback, ajustements
3. Ateliers co-design solutions (avec terrain) : accueil + arrière-office + DIM concevront processus ensemble

**Impact : Dialogue capacitant peut passer 2→4/5 avec gouvernance formelle + réunions coordination + ateliers co-design.**

## 8. DIALOGUE CAPACITANT DURABLE

### Conditions fondatrices (pré-condition succès transformation)

#### A. Engagement visibilité Direction Générale

**Condition** : DG reconnaît désalignement comme problème système (pas incompétence personnel), engage visibilité et arbitrages critiques.

**Actions spécifiques** :
1. Valider "sécurité patient prime sur programme opératoire" (arbitrage urgent/administratif)
2. Allouer budget IT prioritaire refonte ergonomie SIH (investissement vs. coûts cachés)
3. Créer rôle Manager Circuit (reporte DG directement) ; nommer responsable transverse
4. Parrainer Commission urgence/administratif (arbitrage formel tension)

**Implication** : Sans engagement DG, Quick Wins déployés en silo restent ; transformation durable impossible.

---

#### B. Responsabilité métiers explicite (propriétaires flux)

**Condition** : Chaque métier accepte propriété sa portion flux, mais reconnaît interdépendances.

**Admission** :
- Propriétaire : Directeur admission + Manager Circuit
- Responsabilité : F11 créée, F16 activée j+1, complétude j+1 atteinte
- Dépendance acceptée : impact si F16 non-complétée bloque Pharmacie F17/F18

**Pharmacie** :
- Propriétaire : Directeur pharmacie + Manager Circuit
- Responsabilité : Dispensation tracée, imputation j+0-1
- Dépendance acceptée : si admission incomplète → impact imputation impossible

**Bloc** :
- Propriétaire : Chef bloc + Manager Circuit
- Responsabilité : Actes validés j+1, planning SIH synchrone
- Dépendance acceptée : si actes validés tard → impact facturation retardée

**Implication** : Propriété claire + acceptation interdépendances = base dialogue constructif.

---

#### C. Transparence données obligatoire (tableau de bord accessible tous)

**Condition** : Données SIH (complétude, délais, facturation, actes validés, planification) visibles tous quotidiennement.

**Tableau de bord transverse** :
- % admissions complétude j+0 vs. j+1 (cible: 100% j+1)
- Délai F11→F16 (cible: j+1)
- Délai imputation pharmacie (cible: j+0-1)
- Délai acte validation (cible: j+1)
- Délai facturation (cible: j+3 max)
- Taux rejet facturation (cible: <2%)
- Planning bloc respect (cible: 90%+)

**Fréquence mise à jour** : Quotidienne (dashboarding BI temps réel)

**Accès** : Tous (équipes + management + direction)

**Implication** : Transparence = feedback loops réactivées → chacun voit impact actions → motivation amélioration.

---

#### D. Arbitrage formel tensions non-résolvables localement

**Condition** : Créer Commission urgence/administratif (mensuelle CODIR) arbitrant tensions formellement.

**Sujets Commission** :
1. Urgence vs. complétude dossier : "En urgence, quelle complétude minimale j+0 acceptable ? Quand rattrapage j+1 OK ?"
2. Urgence vs. validation BPPH : "En urgence chirurgicale, validation orale documentée acceptable ? Si oui, conditions ?"
3. Planning bloc vs. urgences intercurrentes : "Quand peut-on déprogrammer urgent ? Qui décide ? Comment tracé ?"
4. Consentement urgence : "En urgence vitale, consentement oral tracé j+1 acceptable ? Protocole ?"

**Résultats Commission** → intégrés Golden Process documentation

**Fréquence** : Mensuelle (jour fixe, sacraliser calendrier)

**Sponsor** : DG (sponsorship visible = légitimité)

**Implication** : Arbitrage formel = plus de chaos local ; chacun sait règles ; contournements formellement justifiés vs. tolérés.

## 13. CONCLUSION

### Synthèse état des lieux consolidée

Le SIH présente un **désalignement systémique critique** entre processus formel et pratiques réelles, affectant les trois modules critiques (Admission, Pharmacie, Bloc) et leur chaîne de valeur intégrée.

**Causes racines convergentes** :
1. **Organisationnelles** : Fragmentation responsabilités, absence propriétaire transverse, absence arbitrage formel urgent/administratif
2. **Techniques** : Architecture SIH cloisonnée, paramétrage incomplet, interfaces manuelles, ergonomie inadaptée
3. **Fonctionnelles** : Processus linéaires rigides sans variantes urgence, validations finales au lieu progressives, pas checkpoints intermédiaires
4. **Humaines** : Contournements rationnels face outils inadaptés, culture acceptation imparfait, absence feedback loops
5. **Gouvernance** : Pas KPI transverses, pas escalade formelle, gestion réactive post-incident

**Impacts chiffrés** :
- Surcharge opérationnelle : 220-430 k€/an (doubles saisies, régularisations, retards facturation)
- Retards facturation : 5-15 jours vs. 1-3 cible = impact trésorerie 100-200 k€/an
- Risques légaux latents : Consentement absent (RGPD), DMI non-tracé (matériovigilance), stupéfiants pas registre (légal), check-list orale (HAS)
- **Total impact : 340-670 k€/an + risques réglementaires majeurs**

### Message clé à la direction

**Le désalignement SIH n'est pas un problème technique, c'est un problème de gouvernance et de design système.**

Les équipes adaptent localement face à outils inadaptés et processus rigides. L'amélioration nécessite :
1. **Arbitrages clairs** (urgent vs. administratif formalisé)

3. **Outillage SIH adapté** (interfaces ergonomiques, modes dégradés, variantes urgence)
4. **Transparence données** (feedback loops réactivées)
5. **Dialogue capacitant** (multi-disciplinaire régulier)
